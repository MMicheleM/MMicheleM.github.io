Regex

(Evidently there aren't any funny regex cartoons out there... ergo cute overload.)

Defined

Regex - short for Regular Expression - is a special sequence of characters that allows you to match/find other strings or sets of strings. They also allow you to extract the portions that match.

One of the beautiful things about Regex is that they're ubiquitous - they work across all languages and any capable text editor.

Within the scope of Dev Bootcamp it's also a wonderful thing that there is a Regexp class.

Syntax

The basic Ruby syntax is /pattern/ or %r{pattern} You can also search using the Regexp.new constructor.

The =~ operator is Ruby's basic pattern-matching operator. If a match is found the operation returns the index of the first match in the string otherwise it returns nil.

Metacharacters and Escapes

Metacharacters have a specific meaning when appearing in a pattern. To match them literally many of them must be backslash escaped. Ruby Metacharacters requiring an escape are:

  • (, )
  • [, ]
  • {, }
  • .
  • ?
  • +
  • *

Anchors are a type of metacharacter that match the zero-width positions between characters, anchoring the match to a specific position.

  • ^ matches beginning of the line
  • $ matches end of line
  • \A matches beginning of string
  • \Z matches end of string (if string ends with a new line it matches just before the new line)
  • \z matches end of string
  • \G matches point where last match finished
  • \b matches word boundaries when outside brackets; backspace when inside them
  • \B matches non-word boundaries
I got a bit lost with the following ones and so am listing but not trying to define them.
  • (?=pat) is positive look ahead assertion
  • (?!pat) is negative look ahead assertion
  • (? =pat) is positive look back (minus the space after the )
  • (? !pat)" is negative look back (minus the space after the )

Repetition

  • * stands for zero or more times
  • + stands for one or more times
  • ? zero or one times
  • {n} stands for exactly n times
  • {,m} stands for m or less times
  • {n,m} stands for at least n and at most m times

Examples

Delete empty lines: /^\s*\n/

The caret (^) stands for the start of the line - indicating we'd like to start the search from the beginning of the line. The \s combination represents whitespace The * says that we are looking for 0 or more of the whitespaces And the \n is, of course, our special character for a newline.

As it stands the above example will read through whatever text you have it search on from the beginning of each line through to the end of the line. Depending on what you tell it to (vis-a-vis replacement) it will either be modified as instructed or it will remove all empty lines (if you replace it with nothing)

Remove spaces from beginning of the line: /^ +/ (or /^\s+/)

A variation on the above would be to remove any/all leading white spaces from lines. You can either use a literal space or the \s to indicate the white space.

So we know the ^ is start of line (and it's $ if you're looking to match end of line) The space or \s is our white space And the + looks for one or more of the previous tokens (in this case a white space)
