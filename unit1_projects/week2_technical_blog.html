Classes vs. IDs

The first question to ask yourself is: are you dealing with a one to one situation (ID) or a one to many (class).

Perhaps the easiest way to think about it is this: You are (hopefully!) the only one with your ID but there are a bunch of us in this class/cohort. Therefore if you wished to apply your unique style to yourself you'd select ID however if you wanted to share your fabulous style with all of us you'd want to select class.

They both:

  • Define a special case for an element
  • Behave identically with CSS
  • Can make your head hurt when you're first learning about all this
  • Allow you to make things pretty

Where they differ:

  • Classes allow you to write cleaner code (no styling redundancies needed)
  • IDs can be referenced in a URL and the browser will attempt to scroll to that area

Syntax

In HTML you'll use similar syntax. It'll either be id = or it'll be class =

In CSS you'll being your ID with a hashtag # and your class with a period (or full stop) .
